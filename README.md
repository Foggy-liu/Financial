# README

## 项目概述

本项目是一个端到端的金融新闻标题情感分析系统，能够自动将新闻标题分类为“负面/中性/正面”三种情感类别。系统独立完成了从原始文本分词、向量化、降维到模型训练与对比的全流程特征工程，并通过系统性对比实验与过采样优化，深入探究了样本量对模型表现的影响，最终交付可用于生产环境的情感分类模型。

**项目周期**：2025.05 - 2025.06  
**核心任务**：构建三分类情感分析系统，完成从原始数据到模型交付的全流程

## 一、数据预处理与特征工程

### 1.1 原始数据

- **数据来源**：金融新闻标题，仅包含文本与情感标签
- **标签分布**：0-负面、1-中性、2-正面
- **类别不平衡**：负面(12.5%)、中性(28.1%)、正面(59.4%)

### 1.2 文本向量化流程

| 步骤 | 操作             | 说明                                       |
| ---- | ---------------- | ------------------------------------------ |
| 1    | 分词统计         | 计算平均分词量为13个，作为统一长度基准     |
| 2    | TF-IDF权重筛选   | 筛选关键词语，降低常见词干扰               |
| 3    | 固定长度填充     | 不足补0，超出截断，统一为13词              |
| 4    | Word2Vec词向量化 | 13词 × 50维 = **650维**特征向量            |
| 5    | 数据集划分       | 训练集:测试集 = 8:2                        |
| 6    | 标准化           | 对训练集进行标准化，测试集使用相同变换矩阵 |
| 7    | PCA降维          | 降至100维，保留98%以上信息量               |

### 1.3 降维策略验证

为确定最优降维维度，绘制累计方差曲线进行分析：

- **拐点识别**：80维为信息量拐点
- **对比实验**：80维 vs 100维
- **最终选择**：**100维**，在降维效果与信息保留之间取得平衡

## 二、多模型对比与初步结论

### 2.1 算法选择策略

选取四类代表性算法，覆盖不同建模思想：

| 算法           | 类型         | 选择理由                                                     |
| -------------- | ------------ | ------------------------------------------------------------ |
| **逻辑回归**   | 线性模型     | 数据经PCA降维后为100维线性组合特征，线性模型理论上应表现良好；作为基准模型，简单快速、可解释性强，原生支持多分类、概率输出和类别权重，与小样本场景匹配 |
| **随机森林**   | Bagging集成  | 代表并行集成思想，验证多模型投票机制在情感分类任务上的表现；与XGBoost形成“并行 vs 串行”的集成策略对比，用于揭示不同集成方式对分类效果的影响 |
| **XGBoost**    | Boosting集成 | 代表串行集成思想，通过迭代聚焦难分样本，精度上限高；竞赛常用，用于验证梯度提升框架在三分类任务上的潜力，与随机森林对比不同集成策略的优劣 |
| **朴素贝叶斯** | 概率模型     | 检验特征独立性假设在PCA线性组合特征上的适用性；与判别式模型（逻辑回归、随机森林、XGBoost）形成对比，拓展算法视野，验证概率生成式模型在此任务上的表现 |

### 2.2 评估指标体系

针对类别不平衡问题，构建多维评估体系：

| 指标           | 用途     | 选择理由                           |
| -------------- | -------- | ---------------------------------- |
| **F1宏观平均** | 核心指标 | 给每个类别同等权重，适合不平衡数据 |
| AUC分数        | 辅助指标 | 衡量模型整体区分能力，不受阈值影响 |
| 准确率         | 辅助指标 | 直观易懂，但会受多数类主导         |
| 各类别F1       | 分析指标 | 观察每个类别的具体表现             |

### 2.3 初步实验结果（无过采样）

使用原始训练集（3869条）进行5折交叉验证：

| 模型       | F1宏观平均 | 权重策略 | 表现排名 |
| :--------- | :--------- | :------- | :------- |
| 逻辑回归   | **0.4362** | 平衡权重 | 第1名    |
| XGBoost    | 0.4159     | 平衡权重 | 第2名    |
| 朴素贝叶斯 | 0.4148     | 无权重   | 第3名    |
| 随机森林   | 0.3889     | 无权重   | 第4名    |

**初步结论**：PCA线性降维使数据更适合线性模型，逻辑回归在小样本下表现更稳健。

## 三、过采样实验与结论修正

### 3.1 SMOTE过采样策略

为验证样本量对模型表现的影响，引入SMOTE过采样：

- **扩充后训练集**：6891条（三类样本平衡）
- **目的**：消除类别不平衡，探究纯样本量效应

### 3.2 过采样后实验结果

重新进行5折交叉验证与网格搜索：

| 模型        | F1宏观平均 | 变化       |
| ----------- | ---------- | ---------- |
| **XGBoost** | **0.4550** | ↑ 提升     |
| 随机森林    | 0.4371     | ↑ 显著提升 |
| 逻辑回归    | 0.4022     | ↓ 下降     |

### 3.3 核心发现：样本量与模型表现的关系

| 样本量               | 最佳模型 | 原因分析                               |
| -------------------- | -------- | -------------------------------------- |
| **小样本（3869条）** | 逻辑回归 | 线性模型更稳健，树模型易过拟合         |
| **大样本（6891条）** | XGBoost  | 树模型非线性优势得以释放，反超线性模型 |

**关键结论**：样本量是决定模型表现的关键因素，过采样不仅解决了类别不平衡，更揭示了不同模型对数据规模的适应性差异。

## 六、项目总结

### 6.1 核心成果

1. **完整特征工程链路**：从原始文本到100维特征向量，验证了分词、Word2Vec、PCA降维的可行性
2. **系统性模型对比**：覆盖线性模型、集成学习、概率模型四大类算法
3. **过采样关键发现**：揭示了样本量决定模型选择的关键规律——小样本选线性模型，大样本选树模型
4. **可交付模型**：XGBoost最终模型，测试集AUC=0.6325，F1宏观平均=0.4550

### 6.2 改进方向

| 方向              | 说明                             |
| ----------------- | -------------------------------- |
| 增加负面/中性样本 | 收集更多少数类数据，提升识别能力 |
| 尝试深度学习      | 数据量充足时可尝试LSTM、BERT等   |
| 阈值调优          | 针对不同类别设置差异化分类阈值   |
| 特征工程优化      | 尝试其他降维方法或保留更高维度   |
